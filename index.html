Wire Mock Day 1 (Step by Step)

版权声明：本文为博主原创文章，未经博主允许不得转载。

1.Environment


1.Download wiremock-standalone jar:wiremock-standalone JAR

2.Open cmd, cd to the path where the wiremock-standalone jar located, execute command
java -jar wiremock-standalone-2.5.1.jar

3.Two folders mappings and __files will be generated.

2.Your first mock api


1.Put json file inside mappings folder, i.e

demo1.json
{“request”:{“method”:”GET”,”url”:”/api/wiremocktest1”},”response”:{“status”:200,”body”:”Hello World!\nHello World”}}

Note:Newline character is NOT REQUIRED inside the json file

2.Execute the command java -jar wiremock-standalone-2.5.1.jar

3.Open http://localhost:8080/api/wiremocktest1 (this url is inside the demo1.json file) in browser.

3.Return json in response

1.json is not alowed in body, write a new json file and put in __files folder. i.e

myJsonBody.json
{“CaseId”:123,”WhateverId”:”abc”}

2.The json file inside mappings folder will be
demo2.json
{“request”:{“method”:”GET”,”url”:”/api/wiremocktest2”},”response”:{“status”:200,”bodyFileName”:”myJsonBody.json”}}

Note:response body will be change to bodyFileName

9.Any change in demo2.json,java -jar wiremock-standalone-2.5.1.jar is required to execute. Change in myJsonBody.json will show immediately without executing the command.
